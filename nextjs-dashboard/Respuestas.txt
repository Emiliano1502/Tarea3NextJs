1¿Qué es Next.js y por qué se dice que es el framework de React más popular?
Next.js es un framework de React que permite construir aplicaciones con características,
como serian el renderizado en el servidor , optimizaciones automáticas y rutas de API
Se considera el más popular porque facilita enormemente el desarrollo de aplicaciones
completas y con cara y ojos, ofreciendo herramientas que React por sí solo no tiene, como el
fetching de datos desde el servidor y los React Server Components
2 ¿Cuál es el comando exacto con npx para inicializar un proyecto de Next.js asegurando usar
la última versión?
El comando exacto es: npx create-next-app@latest
3 ¿Cuál es el comando utilizado para levantar el entorno de desarrollo y en qué puerto se
ejecuta por defecto?
Es npm run dev y se ejecuta por en el puerto 3000
4 ¿Qué función cumple el comando npm run build en un proyecto de Next.js?
Este comando se utiliza para crear y empaquetar la aplicación para pasarla a producción
5 Describe para que sirven las carpetas, public y app
public sire para almacenar todos los archivos estáticos que la web servirá directamente como
imágenes, vídeos o audios, mientras que app es la carpeta más importante, donde se gestiona
el contenido de la app mediante archivos page.tsx y donde reside el código principal de las
rutas y componentes
6 ¿Que es un CCS MODULE y como se usan?
Es una forma de cargar estilos donde el archivo debe terminar en .module.css, se usan
importando los estilos como un objeto en el archivo de TypeScript y aplicándolos a través de
sus propiedades como un className={styles.shape}
7 ¿Que hace el componente Layout.tsx ?
Su función es envolver las páginas o rutas para que muestre este
8 ¿Que es el enrutamiento ?
Es poner las rutas de la aplicación como una estructura de carpetas dentro de si misma para
manejar los contenidos de la app
9 ¿Cual es la diferencia entre Image e img?
img es la etiqueta HTML, y en diferencia Image de Next.js ofrece múltiples optimizaciones
automáticas, por ejemplo, evita saltos de diseño al requerir ancho y alto, y aplica Lazy Load
por defecto
10 ¿Que es el Lazy Load?
Es una forma de optimización para la app que consiste en no cargar un recurso o imagen
hasta que sea necesario o esté a punto de aparecer en la pantalla del usuario
11 Cual es el funcionamiento del siguiente codigo, en que momento se muestra cada imagen
El código es para 2 clases de imágenes, la primera imagen (hero-desktop.png) tiene la clase
hidden md:block, ósea está oculta por defecto y solo se muestra en computadoras, mientras
que la segunda (hero-mobile.png) tiene la clase block md:hidden, lo que indica que es visible
por defecto en celulares, pero se oculta en pantallas medianas o grandes.
12 ¿Como funciona el enrutamiento en Next JS ?
Las rutas de la aplicación se definen mediante la estructura de carpetas dentro de la carpeta
app
13 ¿Que es una ruta en el contexto web y donde las hemos visto en la clase ?
Es la parte de la dirección URL que indica a qué sección o recurso específico de la página se
quiere acceder, las hemos usado cuando creamos en vercel una app y de ahí buscábamos
algo de nuestra base de datos

Es el sistema de enrutamiento basado en la estructura de archivos y carpetas
17 ¿Como funcionan las rutas anidadas?
Funcionan simplemente metiendo una carpeta dentro de otra dentro del directorio app,. Cada
nivel de carpeta representa un segmento de la URL final.
18 ¿Investiga por que es necesario esto ?
Es necesario porque permite que ese archivo comparta su funcionalidad con el resto de la app
19 ¿Para que sirve el componente Link?
se utiliza para realizar navegaciones por parte del cliente de forma fluida
20 ¿Que se carga cuando usamos <a> y que cuando se usa <link> ?
Con <a>: Se realiza una recarga completa de la página, con <Link>: Next.js solo carga lo
mínimo necesario para mostrar la nueva rut
